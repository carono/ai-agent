# ai-agent

Мы разрабатываем **набор универсальных агентов для автоматизации разработки** — переиспользуемый шаблон, который подключается к любому проекту.

## Суть

Агенты автоматизируют стандартный цикл разработки (обсуждение → разработка → код-ревью), адаптируясь под воркфлоу конкретного проекта через `docs/WORKFLOW.md`. Не привязаны к конкретному трекеру (GitLab, GitHub, Jira и т.д.) — вся специфика описана в документах проекта, которые агенты читают при запуске.

## Агенты

- **setup** — одноразовый: расспрашивает пользователя о воркфлоу и создаёт `docs/WORKFLOW.md`, `PROJECT.md`, `TECH.md`
- **discussion** — уточняет требования по задачам, переводит в "готово к разработке"
- **worker** — реализует задачи, создаёт код-ревью
- **reviewer** — разбирает замечания к коду, закрывает задачи
- **checker** — генерирует `scripts/check.sh`: скрипт выбора следующего агента по текущему состоянию трекера
- **doc (agents-doc-updater)** — актуализирует человекочитаемую документацию по агентам

## Распространение

Шаблон публикуется как репозиторий `carono/workflow`. Подключается к проекту через:
```
npx degit carono/workflow .claude/agents/carono
```

## Принцип

Агенты не хардкодят названия статусов, API, инструменты. Всё читается из `docs/WORKFLOW.md` конкретного проекта в рантайме.

## Структура проекта

- **Исходники** — `src/`
- **Собранный результат** — `dist/<platform>/`
- **Правило**: правки вносятся ТОЛЬКО в `src/`, файлы в `dist/` никогда не редактируются напрямую — используются только для оценки результата сборки
