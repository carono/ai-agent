# Знания о проекте

Этот файл накапливает бизнес-контекст и особенности проекта. Агенты читают его для понимания предметной области и дополняют когда обнаруживают новую информацию.

---

## Что делает система

Универсальный набор переиспользуемых агентов для автоматизации цикла разработки (обсуждение → разработка → код-ревью). Шаблон подключается к любому проекту — разработчик описывает свой воркфлоу в документах, агенты читают их при запуске и работают без доработки.

---

## Модули и компоненты

- **Агенты** (`src/agents/`) — discussion, worker, reviewer, checker, doc
- **Правила** (`src/rules/`) — инструкции и ограничения для агентов
- **Шаблоны** (`src/templates/`) — шаблоны документов
- **Инструменты** (`src/tools/`) — утилиты для работы агентов
- **Скрипты сборки** (`src/scripts/`) — `build.py` собирает `dist/` из `src/` с шаблонизацией
- **Собранные артефакты** (`dist/`) — готовые пакеты для claude-code и opencode

---

## Интеграции с внешними системами

- **YouGile** — трекер задач (MCP-сервер `yougile`)
- **GitHub** — хостинг кода (репозиторий `carono/workflow`)

---

## Бизнес-правила и ограничения

- Универсальность важнее удобства — специфика проекта живёт в документах проекта, а не в коде агентов
- Агент — читатель документов, не исполнитель хардкода
- Минимальный барьер входа: одна команда подключения
- Ценность в замкнутом цикле, а не в отдельных агентах
- Документация по проекту дополняется автоматически по мере работы агентов

---

## Термины и глоссарий

| Термин | Значение |
|--------|----------|
| Воркфлоу | Набор правил работы с трекером и кодом конкретного проекта |
| Шаблонизация | Подстановка содержимого файлов из `src/rules/` и `src/templates/` в агентов при сборке |
| Цикл | Обсуждение → Разработка → Ревью → Закрытие |

---

## Инструкция по дополнению

Когда в ходе обсуждения задачи или разработки выясняется новая информация о проекте — добавь её в соответствующий раздел. Не дублируй то, что уже есть.
