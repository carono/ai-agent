# Воркфлоу проекта

Этот документ — операционный мануал для агентов. Здесь описано как взаимодействовать с трекером задач, менять статусы, работать с кодом в данном конкретном проекте.

---

## Трекер задач

**Сервис:** YouGile
**Инстанс:** https://ru.yougile.com
**Проект:** WORKFLOW
**Доска:** разработка

### Чтение конфигурации

MCP-сервер `yougile` настроен в `.mcp.json`. Credentials (API-ключ YouGile) хранятся в env-переменной `YOUGILE_API_KEY` внутри `.mcp.json`. Файл `.mcp.json` добавлен в `.gitignore`.

---

## Жизненный цикл задачи

Задачи проходят через колонки доски в порядке:

```
Задачи → Обсуждение → Разработка → Готово
```

### Статусы задачи

| Статус | Значение в трекере | Кто переводит |
|--------|--------------------|---------------|
| Новая | Колонка «Задачи» | Человек (создаёт) |
| В обсуждении | Колонка «Обсуждение» | Человек или бот (переносит для уточнения) |
| В разработке | Колонка «Разработка» | Бот discussion (когда детали выяснены) |
| Готово | Колонка «Готово» | Бот worker (после закрытия MR) |

## Операции с задачами

Используемый MCP-сервер: `yougile`. Конкретные имена инструментов и параметры **не указывай** — агенты определяют их самостоятельно из своего контекста. Описывай только что нужно сделать.

### Проверить репозиторий

Прочитать `.mcp.json`, убедиться что сервер `yougile` настроен и доступен.

### Найти задачи в нужном статусе

Отфильтровать задачи по колонке доски «разработка» проекта WORKFLOW. Колонка соответствует статусу задачи.

### Прочитать задачу

Получить детали задачи: название, описание, вложения, текущую колонку.

### Прочитать переписку задачи

Получить все комментарии к задаче для анализа обсуждений и замечаний по MR.

### Написать сообщение к задаче

Добавить комментарий к задаче — например, результат обсуждения, ссылку на MR, отчёт о выполнении.

### Сменить статус задачи

Переместить задачу в нужную колонку доски «разработка»:
- Из «Обсуждение» в «Разработка» — когда все детали выяснены
- Из «Разработка» в «Готово» — когда MR закрыт

## Работа с кодом

### Ветки

- Создавать от: `master`
- Формат имени: *(уточнить)*
- Целевая ветка для код-ревью: `master`

### Создание код-ревью

Платформа: GitHub (репозиторий `carono/workflow`). MR открывается в ветку `master`. В описании MR обязательно указывать полную ссылку на задачу в YouGile.

### Поиск и чтение код-ревью

Использовать MCP-сервер GitHub для поиска и чтения MR/PR по репозиторию.

### Формат коммита

В сообщении коммита обязательно:
1. **Полная ссылка на задачу** в YouGile
2. Человекопонятное описание того, что было сделано
