# Технический мануал

Этот файл содержит технические детали проекта. Агент-разработчик читает его перед реализацией задачи и дополняет когда обнаруживает новые паттерны или технические особенности.

---

## Стек

- **Язык:** Python 3 (скрипты сборки), Markdown (документация агентов)
- **Платформа:** Node.js/npm (управление скриптами)
- **Сборка:** `src/scripts/build.py` — собирает `dist/` из `src/` с шаблонизацией переменных `{{VARIABLE}}`
- **Трекер:** YouGile (MCP-сервер `yougile`)
- **VCS:** Git, хостинг GitHub (`carono/workflow`)

---

## Структура и документация кода

```
src/
  agents/       — исходники агентов (discussion, worker, reviewer, checker, doc)
  rules/        — правила и ограничения для агентов
  templates/    — шаблоны документов
  tools/        — утилиты для агентов
  scripts/      — build.py (сборка), watch.py (горячая пересборка)
  logs/         — логи (в .gitignore)
dist/
  claude-code/  — собранный пакет для Claude Code
  opencode/     — собранный пакет для OpenCode
docs/           — проектная документация
workflow/       — конфигурация воркфлоу по ботам
```

### Шаблонизация

При сборке `build.py` подставляет содержимое файлов вместо переменных:

| Папка источника | Формат переменной | Пример |
|---|---|---|
| `src/rules/<имя>.md` | `{{RULES_<ИМЯ>}}` | `src/rules/security.md` → `{{RULES_SECURITY}}` |
| `src/templates/<имя>.md` | `{{TEMPLATE_<ИМЯ>}}` | `src/templates/WORKFLOW.md` → `{{TEMPLATE_WORKFLOW}}` |

Новые файлы в `rules/` или `templates/` нужно регистрировать в `build.py::load_template_variables()`.

## Команды разработчика

```bash
# Сборка всех платформ
npm run build

# Сборка конкретной платформы
npm run build:claude-code
npm run build:opencode

# Горячая пересборка (watch)
npm run watch
npm run watch:claude-code
npm run watch:opencode

# Очистка dist/
npm run clean
```

**Правило:** правки вносятся ТОЛЬКО в `src/`, файлы в `dist/` никогда не редактируются напрямую.
