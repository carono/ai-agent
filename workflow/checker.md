Ты агент генерации скрипта выбора агента.

## Задача

Изучи воркфлоу проекта и файлы агентов, выяви условия запуска каждого агента, затем создай или обнови `scripts/check.sh` — скрипт, который определяет какой агент запустить следующим.

## Алгоритм

### Шаг 1 — Изучи воркфлоу

Прочитай `docs/WORKFLOW.md`. Найди и запомни:

1. **Как опросить трекер** — какой инструмент используется (API, CLI, файлы, БД), какая команда/вызов возвращает список задач, назначенных на агента
2. **Какие статусы существуют** — точные идентификаторы (лейблы, поля, теги) каждого статуса
3. **Как читать статус задачи** — из какого поля или лейбла берётся статус
4. **Есть ли стейт-файлы** — пути, структура, назначение

Если `docs/WORKFLOW.md` не существует или не содержит нужной информации — останови работу и сообщи что файл нужно заполнить перед генерацией скрипта.

### Шаг 2 — Изучи агентов

Прочитай все файлы агентов в `workflow/` (кроме `checker.md`, `setup.md`, `doc.md`).

Для каждого агента определи:
- **Условие активации** — в каком статусе должна быть задача чтобы агент её взял
- **Приоритет** — чем позже стадия жизненного цикла задачи, тем выше приоритет (ревью > разработка > обсуждение > новая)
- **Стейт-файл** — использует ли агент стейт для дедупликации

### Шаг 3 — Проверь существующий скрипт

Проверь наличие `scripts/check.sh`:

**Если скрипт существует:**
- Прочитай его
- Сравни условия фильтрации и приоритеты с тем что нашёл в шагах 1–2
- Если всё совпадает — сообщи что скрипт актуален и останови работу
- Если есть расхождения — перейди к шагу 4, обнови только изменившиеся части через Edit

**Если скрипта нет:**
- Создай директорию `scripts/` если её нет
- Перейди к шагу 4, создай скрипт с нуля через Write

### Шаг 4 — Сформируй скрипт

Скрипт должен состоять из следующих блоков в указанном порядке:

---

**Блок 1: Шапка**

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
```

---

**Блок 2: Получение задач**

Сформируй команду получения задач из трекера на основе `docs/WORKFLOW.md`.
Результат должен быть в переменной `ISSUES_JSON` — JSON-массив задач.

Пример структуры (адаптируй под реальный трекер):
```bash
ISSUES_JSON=$(# команда из WORKFLOW.md, возвращающая JSON-массив задач назначенных на агента)
```

Если трекер не JSON — конвертируй на месте через python3 или jq.

---

**Блок 3: Подсчёт задач по агентам**

Для каждого агента — один блок подсчёта. Формируй условие фильтрации точно по логике из шага 2.

```bash
COUNT_<АГЕНТ>=$(python3 -c "
import json, sys
issues = json.loads('''$ISSUES_JSON''')
# условие фильтрации — точное соответствие статусу из WORKFLOW.md
count = sum(1 for i in issues if <условие>)
print(count)
")
```

Правила формирования условий:
- Если агент берёт задачи без статуса: `not i.get('labels') and not i.get('status')`
- Если агент берёт задачи с конкретным лейблом: `'label-name' in [l.lower() for l in i.get('labels', [])]`
- Если агент берёт задачи с конкретным статусом-полем: `i.get('status') == 'value'`
- Адаптируй под реальную структуру JSON из трекера

---

**Блок 4: Дедупликация через стейт (только для агентов со стейт-файлом)**

Если агент использует стейт-файл (как discussion), добавь проверку:

```bash
# Для агента с дедупликацией — проверяем есть ли новые задачи относительно стейта
COUNT_<АГЕНТ>_NEW=$(python3 -c "
import json, os
state_file = '<путь к стейт-файлу>'
state = {}
if os.path.exists(state_file):
    try: state = json.load(open(state_file))
    except Exception: pass
issues = json.loads('''$ISSUES_JSON''')
filtered = [i for i in issues if <условие агента>]
new_count = sum(
    1 for i in filtered
    if str(i['iid']) not in state.get('issues', {}) or
       i['updated_at'] != state['issues'][str(i['iid'])].get('updated_at')
)
print(new_count)
")
```

---

**Блок 5: Логика приоритетов**

```bash
# Агенты с более поздней стадией — выше приоритет
if [ "$COUNT_<ПОСЛЕДНИЙ_АГЕНТ>" -gt 0 ]; then
    echo "<последний-агент>"
elif [ "$COUNT_<СРЕДНИЙ_АГЕНТ>" -gt 0 ]; then
    echo "<средний-агент>"
elif [ "$COUNT_<ПЕРВЫЙ_АГЕНТ_NEW>" -gt 0 ]; then  # если есть стейт — используй _NEW
    echo "<первый-агент>"
else
    echo "none"
fi
```

Имена агентов в `echo` должны совпадать с `name:` в frontmatter их `.md`-файлов.

---

### Шаг 5 — Выставь права и проверь

```bash
chmod +x scripts/check.sh
bash -n scripts/check.sh  # синтаксическая проверка
```

Если синтаксическая проверка не прошла — исправь и повтори.

## Правила

- Скрипт должен выводить в stdout **только одно слово** — имя агента или `none`
- Не выводи ничего кроме этого — никаких логов, debug-вывода, сообщений
- Все параметры подключения к трекеру (токены, URL) читай из файла конфига или переменных окружения — не хардкодь
- Если в `docs/WORKFLOW.md` описан конфиг-файл — читай параметры из него
- Используй Edit для обновления существующего скрипта, Write для создания нового
- Если структура задач в трекере неочевидна — сделай тестовый запрос через Bash и изучи реальный JSON перед написанием условий фильтрации
- Не меняй блоки 1–2 (шапка, получение задач) при обновлении — только блоки фильтрации и приоритетов, если изменились только агенты
