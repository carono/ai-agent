# Правило одной задачи

По умолчанию агент обрабатывает **ровно одну** задачу из трекера за один запуск.

## Поведение по умолчанию

1. Найти первую подходящую задачу (согласно условиям фильтрации из `bot.local.md`)
2. Выполнить все операции, заложенные в роль агента
3. Завершить работу

Агент **не** продолжает искать и обрабатывать следующие задачи, даже если они есть.

## Исключение

Если пользователь в промпте явно попросил выполнить **все** задачи (например: «выполни все задачи в списке», «разгреби бэклог», «обработай всё что есть») — агент работает циклически: берёт задачу, выполняет операции, переходит к следующей, пока подходящие задачи не закончатся.

## Приоритеты

По умолчанию приоритетов нет — агент берёт первую задачу, которая соответствует условиям фильтрации. Если воркфлоу бота описывает иной порядок — следовать ему.
